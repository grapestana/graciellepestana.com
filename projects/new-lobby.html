01
Problem

We had an issue with mostly Gazeus games because their first launch was made creating the games on more app stylesheets, so many of them didn't bring these game feeling ideas or even the elements that represent these games in real life.

02
Process

We wanted to create a new style for the game that brings a game feel to it, as clickable buttons, animations, and more colourful elements as in Spade's case it is a casino game:


  • We've made a benchmark with the main competitors;
  • Sketches and studies of colours, icons and animations to make fun of playing it;
  • Wireframes to understand which is the best places to access each stage of the game;
  • Prototype to test the new solution internally;
  • Usability Test internally to understand if the solutions help the player to choose the game modes and also to connect with the levels;
  • After the Usability Test results, we've changed some bits of the flow.
03
Solution

After all the UX processes we made the interface following the guidelines and the new art direction of the game, which included:


  • User Interface, with vector (Figma) illustrations and cute/cartoonish;
  • New profiles with data of your evolution in the game in each game mode;
  • New In-Game Options and Settings in the game using the new buttons;
  • New animations for screen loading, button interactions, and gameplay animation when you do a Spades broken action;
  • New game mode separation between the number of points to win, and also options to choose to play Solo (1x1 matches) or with a Partner (2x2 matches);
  • Each game mode had a different diamond and the rules were shown in the game mode;
  • All the animations were made with Unity UGUI animator;
  • All the screens and sprites were made on Figma exported and Implemented on Unity UI.
